## Реализованные алгоритмы и структуры данных

### 1. Префиксные суммы — запрос суммы за период O(1)

Массив `prefix[i]` хранит сумму расходов с 1-го по i-й день.  

```
prefix[i] = prefix[i-1] + daily_totals[i]
```
Запрос суммы за период [A, B]:
```
total = prefix[B] - prefix[A-1]
```
Сложность: **O(1)** на запрос, **O(n)** на построение.

-----

### 2. Линейный поиск дня с максимальным расходом

Проходим по всем дням и сравниваем суммы, ищем максимум.  
Сложность: **O(n)**.

-----

### 3. Сортировка вставками категорий по сумме трат

```python
for i in range(1, len(items)):
    key = items[i]
    j = i - 1
    while j >= 0 and items[j][1] < key[1]:
        items[j + 1] = items[j]
        j -= 1
    items[j + 1] = key
```

Сложность: **O(n²)**.

-----

### 4. Стек для отмены последнего расхода

Используется принцип Last In - First Out (LIFO): при добавление расход кладется на стек(push), при отмене - извлекается с вершины(pop).

Стек выбран потому, что нужно отменять именно последнее действие, а стек дает к нему доступ за O(1).
|Операция  |Сложность|
|----------|---------|
|push      |O(1)     |
|pop       |O(1)     |
|peek      |O(1)     |

-----

### 5. Дерево BST для хранения расходов

Бинарное дерево поиска по составному ключу `(amount, day, category)`.  
Составной ключ позволяет корректно хранить расходы с одинаковой суммой.  
Обход `inorder` возвращает расходы в порядке возрастания суммы.

После операции `undo` дерево полностью пересоздаётся для поддержания
согласованности данных.

> Недостаток обычного BST: при добавлении элементов в отсортированном порядке дерево вырождается в список (O(n) вместо O(log n)).

|Операция           |Средняя сложность|Худшая сложность|
|-------------------|-----------------|----------------|
|insert             |O(log n)         |O(n)            |
|inorder            |O(n)             |O(n)            |
|find_min / find_max|O(log n)         |O(n)            |
